FORM FILTERING LOGIC
═════════════════════════════════════════════════════════════════════════════

  Code in farmer/forms.py:
  
  class CropForm(forms.ModelForm):
      def __init__(self, *args, **kwargs):
          super().__init__(*args, **kwargs)
          # Only show ACTIVE crops
          self.fields['master_crop'].queryset = \
              MasterCrop.objects.filter(is_active=True)

  Result: 
    • Dropdown automatically shows only active crops
    • Currently shows: Potato, Tomato, Rice
    • If admin sets is_active=False, crop disappears
    • Can't select inactive crops

DATA PROTECTION: FOREIGN KEY
═════════════════════════════════════════════════════════════════════════════

  Code in farmer/models.py:
  
  class Crop(models.Model):
      master_crop = ForeignKey(
          'admin_panel.MasterCrop',
          on_delete=models.PROTECT  ← IMPORTANT!
      )

  What PROTECT means:
  ✅ Prevents accidental deletion of crops in use
  ✅ If you try to delete: ProtectedError raised
  ✅ Forces you to use is_active=False instead
  ✅ No data loss ever possible

  Example:
  
    Try: MasterCrop.objects.get(name='Potato').delete()
    
    Error: Cannot delete "Potato" because 
           Farmer A uses it in listing #101

ADMIN MANAGEMENT OPTIONS
═════════════════════════════════════════════════════════════════════════════

  Option 1: VIEW/EDIT CROPS
  ─────────────────────────
  Go to: /admin/admin_panel/mastercrop/
  
  See all crops with:
  ✓ Crop name
  ✓ Category (Vegetables, Grains, etc.)
  ✓ Crop type (Organic, Conventional)
  ✓ Active status
  
  Click any crop to edit

  
  Option 2: TOGGLE ACTIVE STATUS
  ──────────────────────────────
  Go to: /admin/ → Master Crop Templates
  Click: Potato
  Uncheck: Is Active checkbox
  Click: Save
  
  Result: Potato disappears from farmer dropdown
          Existing listings remain visible
          No data loss

  
  Option 3: ADD NEW CROP
  ────────────────────
  Go to: /admin/admin_panel/mastercrop/add/
  
  Fill in:
  • Crop Name: (e.g., "Wheat")
  • Crop Type: (Organic/Conventional/Hybrid)
  • Category: (Vegetables/Fruits/Grains/etc.)
  • Description: (Optional)
  • Generic Image: (Optional)
  • Is Active: (True = show in dropdown)
  
  Click: Save
  
  Result: New crop appears in farmer dropdown immediately

  
  Option 4: CANNOT DELETE
  ──────────────────────
  Why? Foreign key protection prevents it
  
  If you try: Delete button shows error
  
  Instead: Uncheck "Is Active" to hide it
